# Pronunciation is a primer + shadowing + L1 podcasts, not an ASR-graded capability

- **Status:** accepted (2026-06-29)
- **Context:** design grill (`grill-with-docs`, 2026-06-29); reverses the speaking-capability direction in roadmap §F #9 and the `project_speaking_exercise_greenlit` memo. Pairs with `CONTEXT.md` → "Pronunciation primer · Shadowing · Pronunciation podcast". Research base: `docs/research/2026-06-29-speaking-intelligibility-pedagogy.md` (deep-research run; synthesis incomplete — see note).

## Context

Roadmap §F #9 (promoted out of "Deferred", greenlit 2026-06-27) wanted a **speaking capability**: mic → server-side ASR (Google STT `id-ID`) → intelligibility verdict → committed to FSRS. The `speaking` exercise type is already scaffolded but dead (`renderContracts.ts` `capabilityTypes: []`); the product constraint on record (`human-product-and-learning-guide.md:326`) is that speaking "should not commit FSRS state until reliable scoring or self-rating policy is explicitly implemented."

The grill asked the prior question — *is an ASR-graded speaking capability even worth building for Indonesian?* — and concluded no.

## Decision

Pronunciation is taught **without** an ASR grader, **without** FSRS scheduling, and **without** any server-side compute or per-use cost. Three parts:

1. **Pronunciation primer** — an L1-aware reference + *perception* surface (sound/alphabet reference consolidated from lesson 1 + minimal-pair audio to hear the contrasts), living in **Leren** as a sibling of the Affix Trainer. Schema-free.
2. **Shadowing** — hear the native model → say it → optional **record + A/B compare**. Client-only (MediaRecorder + playback), ephemeral, no ASR, no FSRS, no storage.
3. **Two L1-specific pronunciation podcasts** (NL for Dutch speakers, EN for English speakers): a **two-host authored script → multi-voice/multi-language TTS → concatenation**, with native `id-ID` example words woven in. Stored as **one `texts` row with twin audio** (`audio_path` NL + `audio_path_en` EN — the grammar-podcast dual-audio pattern), despite the two scripts not being 1:1 (each targets its own L1's pitfalls). Authored by an LLM around a hand-curated per-L1 pitfall brief + example words from the vetted lesson-1 bank, with a review pass. **Produced as a ONE-OFF, not a reusable pipeline** (staff-engineer review, 2026-06-29): the two MP3s are generated by a throwaway script (parameterize `languageCode`, loop segments through the existing TTS client, concatenate once) or manual synth-and-stitch — *not* a productized `scripts/podcasts/`-style deep module. A reusable generator for two rarely-changing files fails the omission test. Demonstrate sounds *inside* short real words (not isolated phonemes) to sidestep the Chirp3-HD short-word weakness, and hand-check every example clip.

The `speaking` exercise type and the `spoken_production` skill cluster remain **dead surface**.

## Why

Indonesian is **phonetic and tone-free**: spelling→sound is ~1:1, stress is regular, the phoneme inventory is small, and the learner pitfalls form a **closed list** (Dutch: g/w/vowels; English: tapped-r/diphthongs/aspiration/initial-ng). A closed list is **taught once**, not drilled per-word forever, and *intelligible ≈ correct* in a way it is not for English (vowels) or Mandarin (tones). The value of "speaking practice" here is **a model to imitate + the act of speaking** — both delivered by the primer + shadowing on existing TTS — not an automated verdict.

Evidence (research run): ASR transcript-match on read/known-target speech is mechanically accurate (~5% MER) **but** false-rejects accented-but-intelligible L2 speech unevenly by L1 (2%→14%+ MER; native-trained-ASR bias is a replicated finding — arXiv 2503.06924), and pronunciation "has no binary right/wrong, it is a continuous scale" (ACL 2023.findings-emnlp.557). No off-the-shelf phoneme-level tool covers Indonesian (Speechace: 6 dialects, no id; Azure pronunciation assessment: 33 locales, id-ID excluded; ELSA: English-only). Duolingo's own scorer targets **intelligibility, not imitation**. So an ASR grader for Indonesian would mostly emit **demotivating false-rejects for near-zero teaching gain**.

Cost is *not* the reason: at single-learner scale STT/Gemini are free-tier-tiny, and at scale the per-use COGS is a manageable pricing input (≈$1/active-user/month), not a blocker. Cheap is simply not the same as worth it.

## Considered and rejected

- **ASR-graded SRS speaking capability** (the greenlit plan). Rejected: marginal pedagogical value for a phonetic language; false-reject demotivation; would be the app's first per-use COGS + the §6 "reliable scoring" bar is unmet for `id-ID`.
- **Multimodal-LLM (Gemini audio-in) phonetic feedback driving FSRS.** Deferred, not adopted: the only route to "Level 2" articulatory feedback for Indonesian, but unproven and carries a hallucination hazard (confident wrong feedback mistrains). Permitted **later** only as *ungraded enrichment* if validated — never as the FSRS verdict.
- **Two separate `texts` rows for the podcasts.** Rejected (user): the twin one-row dual-audio model is the chosen representation, consistent with the grammar podcast.
- **NotebookLM for the podcasts** (as the grammar pipeline does). Rejected: no control over the exact Indonesian example pronunciation or level — fatal for a *pronunciation* podcast.
- **A productized multi-voice TTS + concatenation pipeline** for the podcasts. Rejected (staff-engineer, 2026-06-29): genuinely net-new machinery (no multi-voice or audio-concatenation exists today — `narrator.ts` is single-voice/single-SSML and throws rather than concat; `tts-client.ts` hardcodes `languageCode: 'id-ID'`) built to regenerate two files that almost never change — fails the omission test. The Chirp3-HD short-word failure also makes automated synthesis the *riskiest* route for the most important content (the isolated examples). Produce the two episodes as a one-off instead.

## Supabase / schema impact

Small, additive. **Correction (data-architect + architect review, 2026-06-30):** `audio_path_en` does **not** exist on `texts` — it exists only on `lessons` (the grammar-podcast dual-audio pattern). This slice therefore **adds `audio_path_en` to `texts`** (`ALTER TABLE indonesian.texts ADD COLUMN IF NOT EXISTS audio_path_en text` in `scripts/migration.sql`) + the matching field on the `Podcast` interface in `textService.ts`, and L1-routes the audio in `Podcast.tsx` (`lang==='en' && audio_path_en ? audio_path_en : audio_path`). Primer + shadowing remain schema-free; shadowing is client-only.

**No `text_kind` discriminator** (decided 2026-06-30): the architect and a code check showed the readers degrade gracefully without one — the Podcasts list hides the CEFR badge when `level` is NULL (`Podcasts.tsx:77`), and audio L1-routing keys off `audio_path_en` presence (story podcasts have none, so they're unaffected). A column for one hand-seeded row fails the omission test. The host scripts are stored in the `transcript_*` columns so the detail tabs stay useful.
