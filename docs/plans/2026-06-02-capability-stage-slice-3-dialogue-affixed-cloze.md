# Capability Stage Redesign — Slice 3: `dialogue_line` + `affixed_form_pair` + item-cloze, DB→DB

> **For Claude:** REQUIRED SUB-SKILL: superpowers:executing-plans. This extends the **shipped** Slice-1 (#99/PR #121) and Slice-2 (#100/PR #124) DB→DB spine to the **last two source kinds still reading staging off disk** (`loader.ts:135,141`). It is a **redesign, not a fold** (ADR 0011/0012/0013). **Read the Slice-1 and Slice-2 plans first** — this reuses their seams (`loadFromDb`, the in-stage generator pattern from `generateItemDistractors.ts`/`generateGrammarExercises.ts`, the idempotent writers, `runCapabilityGate`) and their hard-won live-trial lessons (they are requirements here, not history).

**Goal.** Convert `dialogue_line` cloze, `affixed_form_pair`, and **item vocabulary cloze** to the DB→DB spine: read dialogue lines + morphology pairs from the typed lesson-content tables in the DB, **generate** cloze contexts in-stage (port the `cloze-creator` agent's two contracts), write `dialogue_clozes` / `item_contexts(type='cloze')` / `affixed_form_pairs`, **remove the last two staging-file reads** (`cloze-contexts.ts`, `morphology-patterns.ts`), retire the `capability_artifacts` writes for these kinds, and **relocate the `checkDialogueClozes` gate** from `lint-staging` into the Capability Stage — which **dissolves #126** (L5/7/8's cloze block). Validated by a mandatory live trial on a #126-blocked lesson.

**Architecture.** Same deep-module shape — the database is the entire external interface. The import seam (`loadFromDb`) extends to read `lesson_dialogue_lines` + `lesson_section_affixed_pairs` (+ current cloze/affixed cap state for the delta); generation (LLM, in-stage) produces cloze contexts in two modes; pure projectors fan out to `dialogue_clozes` / `item_contexts` / `affixed_form_pairs`; the idempotent export seam writes typed tables only. **Both cloze tables are KEYED** (`dialogue_clozes` `UNIQUE(capability_id)`; `item_contexts` `onConflict(learning_item_id, source_text)`) — so per-row skip-if-exists works as the safety net, simpler than Slice 2's keyless grammar tables; a per-unit generation gate (per item / per dialogue line) prevents redundant LLM calls.

**Tech Stack.** TypeScript + Bun; Supabase JS (service-role) in `scripts/lib/pipeline/capability-stage/`; `@anthropic-ai/sdk` for in-stage generation (mirroring `enrichPos.ts` / `generateItemDistractors.ts`); Vitest; runtime readers in `src/lib/exercise-content/byKind/{dialogueLine,affixedFormPair}.ts` + `byType/{cloze,clozeMcq}.ts` (LOCKED — already DB-sourced from PRs 1-4).

## Implementation reconciliation (post review round 1 — 2026-06-02 — BINDS over body prose)

The architect + data-architect round-1 reviews caught four issues the first draft got wrong. These corrections BIND:

- **R1 (CRITICAL — data-arch C-1): `item_contexts` has NO unique constraint for the upsert key.** `migration.sql:148-159` has indexes on `(source_lesson_id)` and `(learning_item_id, is_anchor_context)` but **no `UNIQUE(learning_item_id, source_text)`** — so the existing `upsertClozeContext`/`upsertItemAnchorContext` `onConflict: 'learning_item_id,source_text'` would throw Postgres `42P10`. Slice 3 **must add** (idempotent DDL): `CREATE UNIQUE INDEX IF NOT EXISTS item_contexts_item_source_text_unique ON indonesian.item_contexts(learning_item_id, source_text);` + run `make migrate-idempotent-check`. The first draft's "No DDL" claim is WRONG — see corrected Supabase Requirements.
- **R2 (MAJOR — A1 + M-1): the existing writers are NOT skip-if-exists.** `upsertClozeContext` (`adapter.ts:613-632`) is `.upsert(... )` → `DO UPDATE` (overwrites every column); `replaceDialogueClozes` (`adapter.ts:303-372`) is unconditional delete-then-insert. Both contradict the ADR-0011 "reviewed clozes preserved" gate. **Resolution (binds Task 5):** the **per-unit generation gate is the sole idempotency mechanism** — for an already-seeded unit (dialogue line / item), the stage runs **neither the generator nor the writer** (no LLM call, no DB touch). `upsertClozeContext` additionally converts to insert-if-absent (`ignoreDuplicates: true` once R1's unique index exists) as a safety net. `replaceDialogueClozes` is invoked **only** for un-seeded units or under `--regenerate`. A test asserts a second publish mutates zero existing `dialogue_clozes`/`item_contexts(cloze)` rows.
- **R3 (MAJOR — M-2): close the `dialogue_clozes` translation triangle.** Reader `byKind/dialogueLine.ts:62-68` SELECTs `translation_text` (NOT NULL, single-language). PR 6 added nullable `translation_nl`/`translation_en` (`migration.sql:2809-2816`), currently unpopulated. **Resolution (binds Task 4):** the in-stage writer populates **`translation_text` (the NL leg, matching the PR-2 convention) AND `translation_nl` + `translation_en`** — sourced from `lesson_dialogue_lines.translation_nl/en` (PR 6, `migration.sql:2795-2801`), no extra LLM translation needed. The reader switch to `translation_nl/en` with `userLanguage` branching is **out of scope** (named for a later slice); Slice 3 keeps the reader on `translation_text` and guarantees it is populated.
- **R4 (MAJOR — A2): relocate ALL THREE cloze checks.** `lint-staging.ts:951-955` runs `checkClozeContextsFile` (:953), `checkClozeCoverage` (:550/:954), `checkDialogueClozes` (:596/:955). The first draft named only two. Task 8 must relocate/remove all three; `checkClozeContextsFile` validates the `cloze-contexts.ts` file structure and would error on the now-absent file after Task 9. Ownership of deleting the emptied `lint-staging` shell stays with epic #109 — Slice 3 removes its three checks; if they are the last, #109 deletes the shell.
- **R5 (round-2 architect WARNING — resolve at Task 5 build): specify the item-cloze "seeded unit" predicate.** `item_contexts` is **many-rows-per-item** (key `(learning_item_id, source_text)`, no per-item uniqueness), and the LLM-authored carrier `source_text` is **non-deterministic** — so a pre-generation gate keyed on `(learning_item_id, source_text)` can't match before generating (defeating "skip the LLM call"), while keying on item-id-has-any-cloze freezes an item at its first carrier sentence forever. **Decision for Task 5:** the item-cloze generation unit is **the item**, and the design intends a **fixed target count of carrier sentences per item** (e.g. N=the count today); the gate is "item has ≥ N `item_contexts(cloze)` rows → seeded → skip generation." State N explicitly and make the gate count-based, not content-match-based. (Dialogue mode is unaffected — its unit is the dialogue line, keyed 1:1 by `source_line_ref`.)
- **R6 (round-2 data-arch MINOR — operator pre-apply step): dedup-check before the R1 unique index.** `CREATE UNIQUE INDEX … item_contexts(learning_item_id, source_text)` will fail at `make migrate` if the live DB already holds duplicate `(learning_item_id, source_text)` pairs. Before applying: run `SELECT learning_item_id, source_text, count(*) FROM indonesian.item_contexts GROUP BY 1,2 HAVING count(*) > 1;` — if non-empty, dedup first. Recorded in the live-DB grounding section as a required pre-DDL step (cannot be confirmed at authoring — openbrain unavailable).
