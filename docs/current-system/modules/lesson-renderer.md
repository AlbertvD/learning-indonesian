# Lesson renderer — RETIRED (PR 5, 2026-05-25)

The generic page-block render path this spec described — `LessonReader.tsx`,
`blocks/LessonBlockRenderer.tsx`, and `src/lib/lessons/experience.ts`
(`buildLessonExperience`), driven by the `lesson_page_blocks` table — was
**deleted end-to-end in PR 5** (`docs/plans/2026-05-22-data-model-migration.md`
§8). The `lesson_page_blocks` table was dropped.

## What replaced it

**Bespoke per-lesson pages are the sole lesson renderer.** Each lesson has a
hand-authored page at `src/pages/lessons/lesson-<N>/Page.tsx` that imports a
static `content.json` snapshot (generated by `scripts/fetch-lesson-content.ts N`
from `lesson_sections.content`) and renders it with lesson-specific editorial
layout. The pages are registered by lesson UUID in
`src/pages/lessons/registry.tsx`; `src/pages/LessonRouter.tsx` looks the UUID up
and renders the bespoke page (an unregistered UUID is a not-found case — there
is no generic fallback).

Shared, still-live components under `src/components/lessons/` that the bespoke
pages compose:
- `ActivationGate.tsx` — the lesson-activation checkbox (wraps `lib/lessons/activation`).
- `PracticeActions.tsx` — the practice/review CTAs (capability counts via `getLessonCapabilityPracticeSummaryByLessonId`).
- `LessonAudioPlayer.tsx` — whole-lesson audio playback.

`src/pages/LocalPreview.tsx` (`/preview`, gated by `VITE_LOCAL_CONTENT_PREVIEW`)
now renders the real bespoke pages from `content.json` instead of the old
synthetic page-block model.

## Why (ADR 0011 source-of-truth split)

The `lesson_sections.content` blob renders the lesson pages (via the bespoke
`content.json` snapshots); the typed `lesson_sections` tables (PR 6) are the
**capability-stage contract**. Page-block rendering layout (`lesson_page_blocks`)
belonged to neither side once bespoke pages owned rendering, so it was retired
rather than migrated to typed tables (target-architecture Rule #10 — no dead
infrastructure). See `docs/adr/0011-capability-content-is-db-authoritative-after-seeding.md`
and the shipped `2026-05-20-retire-page-blocks-pipeline-phase-1.md` (Phase 1
removed the Stage A writer; PR 5 is the final cut).

## Where the bespoke-page system is specified

There is no single living module spec for the bespoke pages yet (they are
hand-authored per lesson by the `lesson-page-designer` agent). The reader path
is small enough to read directly: `LessonRouter.tsx` → `registry.tsx` →
`src/pages/lessons/lesson-<N>/Page.tsx`. The capability contract those pages'
source content feeds is `docs/current-system/modules/capabilities.md` and the
PR 6 typed `lesson_sections` tables.
